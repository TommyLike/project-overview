# 风险评估 — Claude Code

> **来源**: https://github.com/anthropics/claude-code | **分析日期**: 2026-02-18

## 整体风险：🟡 中等（针对开发者工具采用场景）

Claude Code 由全球资金最雄厚的 AI 公司之一背书，发布持续不断，已实现主流采用。风险是真实存在的，但属于任何专有 SaaS 开发者工具的预期取舍，并非罕见的阻断性风险。需要重点评估的关键风险包括：许可证条款、数据收集、API 成本不可预期性以及供应商锁定。

---

## 许可证风险 ⚠️ — 最重要的发现

**许可证**：© Anthropic PBC. All rights reserved.

**这不是开源软件。** GitHub 代码仓库不包含 Claude Code 的源代码——它包含的是插件、示例、文档链接和 Issue 跟踪。编译后的二进制文件通过安装脚本独立分发。

| 使用场景 | 风险 |
|----------|------|
| 作为开发者工具使用 Claude Code | 🟢 在 Anthropic 服务条款下允许 |
| 将 Claude Code 嵌入你的产品 | 🔴 不允许——保留所有权利 |
| 审计源代码以评估安全性 | 🔴 不可能——源代码未公开 |
| 在本地/内网运行 Claude Code | 🔴 不支持 |
| 分叉或定制二进制文件 | 🔴 不允许 |
| 再分发 | 🔴 不允许 |

**对开发团队的实际影响**：作为*工具*（而非嵌入代码中的库），"保留所有权利"的许可证与使用其他专有开发工具（VS Code 是 MIT，但 Cursor、GitHub Copilot 等众多工具也是专有的）没有本质区别。核心问题是你的组织对专有工具和 Anthropic 服务条款的接受程度。

## 数据收集与隐私风险 🟡

Claude Code 将你的代码和对话发送到 Anthropic 的服务器进行模型推理。

| 数据方面 | 政策 |
|----------|------|
| 代码上传至服务器 | 是——模型推理的必要条件 |
| 数据保留 | 敏感信息保留期有限（依据隐私政策） |
| 训练用途 | 政策声明反馈数据**不用于**模型训练 |
| 企业管控 | AWS Bedrock / Vertex AI 可将数据保留在你的云端 |
| GDPR/合规 | 受 Anthropic 隐私政策和商业服务条款约束 |

**缓解选项**：
- 使用 **AWS Bedrock** 或 **Google Vertex AI** 作为模型提供商——推理流量经过你的云协议，可能满足数据驻留要求。
- 使用**托管网络策略**将出站流量限制在已审批的端点。
- 查阅 Anthropic 完整的[数据使用文档](https://code.claude.com/docs/en/data-usage)。

**企业注意事项**：对于处理高度敏感知识产权、受监管数据（HIPAA、PCI、涉密）或受严格数据主权法律约束的团队，需仔细评估 Anthropic 的合规态势。"使用 Bedrock"是主要的企业缓解措施。

## 供应商锁定风险 🟡

| 方面 | 风险 |
|------|------|
| 模型锁定 | 仅支持 Claude 模型（不支持多模型） |
| 工作流定制 | 插件、Hook 和 CLAUDE.md 可迁移，但技能体系是 Claude Code 专有的 |
| API 成本 | 按 Token 付费——智能体会话可能非常昂贵且难以预算 |
| 定价变更 | Anthropic 可单方面调整 Claude API 定价 |
| 服务连续性 | 完全取决于 Anthropic 持续运营服务 |

**成本不可预期性是最常见的投诉**：对大型代码库进行智能体会话可能消耗数百万 Token。团队反映每月 API 账单因使用模式差异波动较大。团队级别推广前应建立预算管控机制。

## 关键人物 / 组织风险 🟢

对于开源项目，关键人物风险聚焦于个人；而对于 Claude Code，这个问题关乎 Anthropic 作为公司的稳健性：

| 信号 | 评估 |
|------|------|
| Anthropic 融资总额 | 约 77 亿美元以上 |
| 主要投资方 | Amazon（超 40 亿美元）、Google（约 3 亿美元） |
| 营收 | 持续增长的订阅 + API 收入 |
| 团队规模 | 约 3,000 名以上员工（2025 年估算） |
| 被收购风险 | 热门收购目标；收购通常意味着产品延续 |
| 关闭风险 | 鉴于投资规模和营收，极低 |

个人贡献者集中度（`bcherny` 以 70 次提交领先）在组织层面无关紧要——Anthropic 拥有完整的 Claude Code 产品团队支撑。

## 破坏性变更风险 🟡

CHANGELOG 记录了频繁的弃用和移除：

- 遗留 SDK 入口点已移除 → 必须迁移到 `@anthropic-ai/claude-agent-sdk`
- `DEBUG=true` 已移除 → 替换为 `ANTHROPIC_LOG=debug`
- `--print` JSON 输出格式变更（明确标注为破坏性变更）
- 配置选项弃用通知提前 1-2 个版本给出后再移除
- Windows 托管设置路径已迁移

**规律**：破坏性变更大约每 10-20 个版本发生一次，配合 1-2 个版本的弃用通知期。与大多数专有 SaaS 工具相比，通知周期合理。但对 Claude Code 输出/行为有脚本化依赖的团队必须持续关注 CHANGELOG。
