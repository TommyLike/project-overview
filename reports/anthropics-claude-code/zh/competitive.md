## 市场定位：**品类领导者（终端原生 AI 编程智能体）**

## 何时选择 Claude Code

**以下情况推荐选用 Claude Code：**

1. **智能体深度是首要需求** — Claude Code 可跨整个代码库执行多步骤任务，端到端处理 git 工作流，运行 shell 命令。这远超代码补全的能力范畴。
2. **偏好终端原生工作流** — Claude Code 植根于终端，与现有工具集成，而非替换你的 IDE。
3. **使用 AWS Bedrock 或 Google Vertex** — 原生提供商支持意味着可将数据保留在你的云协议内。
4. **MCP 集成至关重要** — Claude Code 的 MCP 支持可接入数据库、API 和内部工具，构建强大的智能体流水线。
5. **Claude 模型质量是核心关切** — 在复杂推理任务上，Claude Opus/Sonnet 在编程基准测试中优于大多数竞争对手。
6. **需要可扩展的插件系统** — Hook、命令、智能体和技能均可定制，以匹配团队工作流。

**以下情况不推荐选用 Claude Code：**

1. **需要开源、可审计的工具** — 所有代码均为专有。Aider 是最佳的开源替代方案。
2. **数据主权是硬性要求且无法通过云端解决** — 即使使用 Bedrock/Vertex，元数据仍流经 Anthropic 系统。不支持本地化部署。
3. **希望避免 API 成本意外** — 智能体会话可快速消耗大量 Token。Copilot（订阅制）或 Cursor（固定月费）的成本更具可预期性。
4. **团队需要多模型灵活性** — Claude Code 锁定 Claude 模型。Cursor 和 Continue.dev 支持多个模型提供商。
5. **预算极为有限** — Aider（BYOK，免费工具）或 Copilot（10 美元/月）的轻量使用成本远低于 Claude Code。

## 竞争趋势

- **GitHub Copilot 正在追加智能体功能** — Microsoft 加速投入 Copilot 的多步骤智能体能力，但在智能体深度上仍落后于 Claude Code。
- **Cursor 在 IDE 优先细分领域势头强劲** — Cursor 增长迅猛，但属于不同产品（完整 IDE 替代 vs. 终端伴侣），两者并非直接竞争。
- **Aider 持续是开源领域的标杆** — 对于需要开源和可审计性的团队，Aider 是明确首选，短期内不会动摇。
- **Claude Code 正在决定性地占领终端/CLI AI 编程细分** — 以每周 650 万的下载量，已完成这一细分市场的格局奠定。
